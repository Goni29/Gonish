# Gonish 내부 가격 기준표

이 문서는 `내부용`입니다.
사이트에 공개하지 않고, 실제 상담과 최종 견적 작성 기준으로만 사용합니다.

## 1. 기본 원칙

- 최소 시작가는 `45만 원` 아래로 내리지 않는다.
- 기본 포함 범위는 `반응형 제작`, `기본 문의 흐름`, `기본 메타 세팅`, `배포 연결`이다.
- `도메인`, `호스팅`, `유료 플러그인`, `외부 결제 수수료`, `사진 촬영`, `전문 카피라이팅`은 별도다.
- 기능 요구가 애매하면 바로 확정가를 주지 말고, 먼저 범위부터 정리한 뒤 본견적을 준다.
- 예약, 회원, 결제처럼 운영 기능이 들어가면 무조건 버퍼를 둔다.

## 4. 계산 방식

1. 프로젝트 유형 기본가를 잡는다.
2. 페이지 규모, 자료 준비도, 일정, 기능 가산을 더한다.
3. 계산된 금액을 `5만 원 단위`로 올림한다.
4. 최종 공개가는 아래 버퍼를 더해 범위로 안내한다.

### 버퍼 규칙

- 계산값이 80만 원 미만이면 `+15만 원`
- 계산값이 80만 이상 150만 미만이면 `+20만 원`
- 계산값이 150만 이상 240만 미만이면 `+30만 원`
- 계산값이 240만 원 이상이면 `+40만 원`
- 기능이 2개 이상이면 `+10만 원` 추가 버퍼
- 페이지가 7개 이상이면 `+10만 원` 추가 버퍼

## 5. 빠른 판단용 기준

### 45만 ~ 70만 원

- 1~3페이지
- 텍스트/사진이 이미 있음
- 기능은 문의 폼 정도
- 디자인도 너무 복잡하지 않음

### 75만 ~ 120만 원

- 4~6페이지
- 브랜드 소개와 서비스 구조 정리가 필요함
- 사례나 갤러리 정도의 추가 화면이 있음
- 리뉴얼 또는 브랜드 홈페이지 성격

### 120만 ~ 180만 원

- 콘텐츠 양이 많음
- 자료 정리가 덜 되어 있음
- 예약, 다국어, 운영 흐름 일부 포함
- 수정 포인트가 중간에 늘 가능성이 있음

### 180만 원 이상

- 로그인, 회원, 결제, 예약 등 실제 기능이 큼
- 페이지가 많고 구조가 복잡함
- 외부 서비스 연동 가능성이 있음
- 일정도 빠르면 더 올린다

## 6. 추천 운영 규칙

- 수정 2회까지 기본 포함
- 큰 방향 변경은 `추가 견적`로 다시 잡는다
- 급한 일정은 무조건 `30만 원 이상` 추가
- 결제/회원/예약은 처음부터 `160만 원부터`로 열어두는 편이 안전하다
- 리뉴얼은 기존 사이트 상태가 심각하면 새 제작처럼 본다

## 8. 최종 견적 작성 팁

- 고객에게는 세부 계산식보다 `총 범위 + 포함/별도 항목 + 일정`을 또렷하게 보여주는 편이 좋다.
- 내부에서는 계산식을 남기고, 외부 견적서에는 `불필요하게 싸 보이는 단가표`는 넣지 않는다.
- 기능 범위가 흔들릴 것 같으면 `예상 범위`로 먼저 보내고, 상담 후 `확정 견적`으로 다시 전달한다.
